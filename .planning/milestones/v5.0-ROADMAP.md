# Roadmap: OpenLearnV2 v5.0 — 核心教学闭环

**里程碑目标：** 将 OpenLearnV2 从"功能框架"升级为"真正可教学的完整平台"，补齐课程内容创作、题库测验、学习路径和进度管理四大核心教学能力缺口。

**前置条件：** v4.0 质量基础与生产就绪（已完成）

---

## Phases 总览

| Phase | 名称 | 主题 | 优先级 |
|-------|------|------|--------|
| 23 | 富文本课程编辑器 | 所见即所得编辑、多媒体嵌入、模板系统 | 🔴 高 |
| 24 | 题库系统增强 | 多题型支持、题库管理、自动组卷、限时考试 | 🔴 高 |
| 25 | 学习路径与解锁机制 | 课程体系化、前置条件解锁、自适应推荐 | 🔴 高 |
| 26 | 学习进度仪表板 | 学生端可视化、教师端数据监控、学习分析 | 🟡 中 |
| 27 | 课程讨论与通知 | 讨论区、公告、推送通知、即时通讯 | 🟡 中 |
| 28 | 实时授课基础 | WebRTC 音视频、屏幕共享、举手发言 | 🟡 中 |

## Phase 23: 富文本课程编辑器

**目标：** 将课程内容从纯文本/Markdown 升级为所见即所得的富文本编辑器，支持图片、视频、数学公式等多媒体嵌入。

**需求：** EDIT-RICH-01 ~ EDIT-RICH-05

### EDIT-RICH-01: 富文本编辑器集成
- **当前问题：** `lessons.content` 存储纯文本/Markdown，AI Agent 生成的课程内容无法呈现丰富格式
- **验收标准：**
  - 集成 TipTap 或 Slate.js 作为富文本编辑器
  - 支持：标题层级、粗/斜体、列表（有序/无序）、引用块、代码块、链接、表格
  - 编辑器内容以 HTML 格式存储在 `lessons.content` 字段
  - 向后兼容现有 Markdown 内容（渲染时自动转换）

### EDIT-RICH-02: 多媒体嵌入
- **当前问题：** 课程内容无法插入图片、视频、音频
- **验收标准：**
  - 图片上传（拖拽/粘贴/选择文件），存储在 VFS 或 uploads 目录
  - 视频嵌入（支持 YouTube/B站 iframe 嵌入 + 本地上传 MP4）
  - 音频嵌入（MP3 上传 + 内联播放器）
  - 文件附件（PDF/PPT 等，提供下载链接）
  - 多媒体文件大小控制在课件上传限制内

### EDIT-RICH-03: 数学公式支持
- **需求：** 理科教学需要数学/化学公式
- **验收标准：**
  - 集成 KaTeX 或 MathJax 渲染 LaTeX 公式
  - 编辑器内提供公式插入工具栏（常用公式模板）
  - 行内公式和块级公式支持

### EDIT-RICH-04: 课程模板系统
- **当前问题：** 教师每次需要从零开始创建课程内容
- **验收标准：**
  - 预设模板：讲授型、讨论型、实验型、复习型
  - 教师可将课程保存为模板
  - 从模板创建新课程（复制结构 + 内容框架）
  - 模板在数据库中存储为 `lesson_templates` 表

### EDIT-RICH-05: AI 辅助内容生成增强
- **当前问题：** AI Agent 只能生成纯文本课程内容
- **验收标准：**
  - AI Agent 可生成富文本格式的课程内容（HTML 结构）
  - AI 可自动检索并嵌入相关图片（基于课程主题）
  - AI 可生成结构化教学大纲 → 自动填充课程各环节

## Phase 24: 题库系统增强

**目标：** 从仅支持基础选择题扩展到多题型题库，支持自动组卷和限时考试。

**需求：** QUIZ-BANK-01 ~ QUIZ-BANK-06

### QUIZ-BANK-01: 题库管理
- **当前问题：** 测验题目内嵌在白板元素中，无独立题库
- **验收标准：**
  - 新建 `question_bank` 表：id, title, type, content(JSON), options, answer, difficulty, knowledge_points, tags, subject, created_by, created_at
  - 题目 CRUD API + 管理界面
  - 支持导入/导出（CSV/JSON 格式）
  - 题目搜索/过滤（按类型、难度、知识点、标签）

### QUIZ-BANK-02: 多题型支持
- **当前问题：** 仅支持选择题
- **验收标准：**
  - 题型：单选题、多选题、判断题、填空题、简答题
  - 编程题（在线代码编辑器 + 测试用例执行）
  - 拖拽匹配题
  - 每个题型有独立的数据结构和渲染组件
  - 简答题支持 AI 辅助评分

### QUIZ-BANK-03: 自动组卷
- **需求：** 教师指定规则，系统自动生成试卷
- **验收标准：**
  - 组卷规则：题目数量、题型分布、难度分布、知识点覆盖
  - AI 辅助选题（自然语言描述 → 自动匹配题库）
  - 随机组卷（防止作弊）
  - 生成试卷预览 + 手动调整

### QUIZ-BANK-04: 限时考试模式
- **当前问题：** 随堂测验无时间限制
- **验收标准：**
  - 设置考试开始/结束时间
  - 倒计时显示（前端 CountdownTimer 已有基础）
  - 时间到自动提交
  - 防作弊：禁止切屏、禁止复制粘贴（可配置）
  - 断线恢复：考试进度保存，重连后可继续

### QUIZ-BANK-05: 测验结果分析
- **验收标准：**
  - 逐题正确率统计
  - 学生答题详情（每题用时、选项分布）
  - 知识点掌握度分析
  - 错题本（学生端个人错题集）
  - 班级对比：平均分、中位数、标准差

### QUIZ-BANK-06: 白板测验集成
- **当前问题：** quiz/quiz-pro 插件为独立插件
- **验收标准：**
  - 白板元素中的测验类型更新为引用题库题目
  - 教师从题库中选择题目插入到课程环节
  - 测验结果自动关联到课程时间线环节

## Phase 25: 学习路径与解锁机制

**目标：** 将零散的课程组织为体系化学习路径，支持基于前置条件的递进式学习。

**需求：** PATH-LEARN-01 ~ PATH-LEARN-04

### PATH-LEARN-01: 课程体系结构
- **当前问题：** 所有课程平级排列，无层次结构
- **验收标准：**
  - 新建 `curriculum` 表：id, title, description, subject, grade_level
  - 课程可归属到"章 → 节 → 课"三级体系
  - 课程列表按体系树展示
  - 支持拖拽排序重排

### PATH-LEARN-02: 前置条件解锁
- **当前问题：** `lessons.progress_mode` 和 `progress_conditions` 字段已定义但未实际使用
- **验收标准：**
  - 激活现有 `progress_mode` 字段：manual（手动）/ sequential（顺序解锁）/ conditional（条件解锁）
  - `progress_conditions` JSON 解析：{ "prerequisite_lessons": ["id1", "id2"], "min_quiz_score": 60, "min_watch_duration": 300 }
  - 前端：未解锁课程显示为灰色锁定状态
  - 解锁时推送通知

### PATH-LEARN-03: 自适应学习推荐
- **需求：** AI 根据学生表现推荐下一步学习内容
- **验收标准：**
  - 基于测验成绩和作业完成情况调整推荐
  - AI Agent 新增 `learning.recommend_next` 工具
  - 学生端首页显示"推荐学习"卡片
  - 薄弱知识点自动推送补充材料

### PATH-LEARN-04: 学习任务与截止日期
- **验收标准：**
  - 教师可为课程/作业/测验设置截止日期
  - 学生端显示待办清单（按截止日期排序）
  - 超时未完成标记 + 教师端逾期名单
  - 学生可标记任务状态（待办/进行中/已完成）

## Phase 26: 学习进度仪表板

**目标：** 为学生和教师提供直观的学习进度可视化，支撑数据驱动的教学决策。

**需求：** DASH-STU-01 ~ DASH-STU-04

### DASH-STU-01: 学生端学习仪表板
- **当前问题：** 学生只能看到课程列表，无进度概览
- **验收标准：**
  - 总体进度环形图（已完成/进行中/未开始）
  - 近期学习活动时间线
  - 待完成作业/测验/截止日期清单
  - 总分/平均分趋势折线图
  - 班级排名（匿名化，仅显示百分位）

### DASH-STU-02: 教师端教学监控面板
- **当前问题：** 教师需逐一查看各学生进度
- **验收标准：**
  - 班级总览：全班进度分布热力图
  - 风险学生预警（低完成率、低分、长期未登录）
  - 课程完成漏斗（各环节完成率）
  - 作业批改进度（待批改数/已批改数）

### DASH-STU-03: 学习行为分析
- **验收标准：**
  - 学习时长统计（日/周/月）
  - 活跃时段分析
  - 薄弱知识点识别
  - 学习投入度评分

### DASH-STU-04: 成绩预警与干预
- **验收标准：**
  - 成绩下降趋势自动预警
  - 教师端一键推送补课材料给预警学生
  - 预警历史追踪

## Phase 27: 课程讨论与通知

**目标：** 建立师生之间、学生之间的沟通桥梁，让学习不再是单向的。

**需求：** COMM-DISC-01 ~ COMM-DISC-04, COMM-NOTIF-01 ~ COMM-NOTIF-02

### COMM-DISC-01: 课程讨论区
- **验收标准：**
  - 新建 `discussions` 表：id, lesson_id, student_id, title, content, parent_id（支持嵌套回复）, created_at
  - 每个课程有独立讨论区
  - 支持 Markdown 格式
  - 教师可置顶/加精/删除帖子
  - 学生可"点赞"问题和回答

### COMM-DISC-02: 全局问答（Q&A）
- **验收标准：**
  - 跨课程的全局问答板块
  - 标签系统（按学科、知识点分类）
  - AI 自动建议已有答案
  - 最佳答案标记（教师认证）

### COMM-NOTIF-01: 通知系统
- **当前问题：** 只有 Socket.IO 即时推送框架，无持久化通知
- **验收标准：**
  - 新建 `notifications` 表：id, user_id, type, title, body, link, read, created_at
  - 通知类型：作业批改完成、新测验发布、讨论回复、成绩更新、课程公告
  - 前端通知铃铛图标 + 未读红点
  - 通知中心页面（分页查看历史）
  - 浏览器 Notification API 推送（需用户授权）

### COMM-NOTIF-02: 课程公告
- **验收标准：**
  - 教师可在课程内发布公告（置顶显示）
  - 公告自动触发全课程学生通知
  - 支持富文本格式

## Phase 28: 实时授课基础

**目标：** 为平台添加基本的实时音视频授课能力，支持教师直播讲课。

**需求：** LIVE-RTC-01 ~ LIVE-RTC-04

### LIVE-RTC-01: WebRTC 音视频直播
- **当前问题：** 无实时音视频，仅文字聊天 + 白板同步
- **验收标准：**
  - 教师端：一键开启直播（摄像头 + 麦克风）
  - 学生端：观看直播流（仅观看，默认静音）
  - 使用 WebRTC + SFU（Selective Forwarding Unit）架构或简单 PeerConnection
  - 教师端可切换摄像头/屏幕共享
  - 延迟控制在 500ms 以内

### LIVE-RTC-02: 屏幕/课件共享
- **验收标准：**
  - 教师可共享整个屏幕或单个应用窗口
  - 共享时可同时展示摄像头小窗（画中画）
  - 课件与屏幕同步（教师翻页学生跟随）

### LIVE-RTC-03: 互动控制
- **验收标准：**
  - 学生"举手"（教师在名单中看到举手列表）
  - 教师授权单个学生开麦发言
  - 教师可全体静音
  - 在线状态指示（小绿点）

### LIVE-RTC-04: 录制与回放
- **验收标准：**
  - 教师可录制直播（音视频 + 白板 + 课件操作）
  - 录制文件存储在服务器或上传到云存储
  - 学生可回放课程录像
  - 录制列表与课程关联
