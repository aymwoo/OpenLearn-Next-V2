## Phase 23: 富文本课程编辑器

### EDIT-RICH-01: 富文本编辑器集成

- **描述：** 将课程内容从纯文本/Markdown 升级为所见即所得的 TipTap 编辑器。
- **用户故事：** 作为一名教师，我希望能在课程中排版出有层级结构的教学内容，就像使用 Word 一样简单。
- **验收标准：**
  1. 集成 TipTap 编辑器组件
  2. 工具栏支持：标题（H1-H4）、粗体、斜体、下划线、删除线、有序/无序列表、引用块、代码块、链接、表格
  3. 编辑器内容以 HTML 格式存入 `lessons.content`
  4. 现有纯文本/Markdown 内容渲染时自动兼容
  5. 支持撤销/重做（Ctrl+Z/Y）
- **技术选型：** `@tiptap/react` + `@tiptap/starter-kit` + `@tiptap/extension-table`
- **影响文件：** `src/App.tsx`（课程编辑面板）、`server.ts`（lesson.update handler）

### EDIT-RICH-02: 多媒体嵌入

- **描述：** 课程内容中支持插入图片、视频、音频和文件附件。
- **验收标准：**
  1. 图片：支持拖拽/粘贴/本地上传，自动存储到 VFS
  2. 视频：支持 YouTube/B站 URL 嵌入（iframe）+ 本地上传 MP4（<50MB）
  3. 音频：MP3 上传 + 内联 `<audio>` 播放器
  4. 附件：PDF/PPT/ZIP 等文件上传，提供下载链接
  5. TipTap 自定义扩展节点：ImageNode, VideoNode, AudioNode, AttachmentNode
- **影响文件：** 前端编辑器组件、VFS 存储逻辑
- **新依赖：** 无额外依赖（TipTap 扩展即可）

### EDIT-RICH-03: 数学公式支持

- **描述：** 支持 LaTeX 数学公式渲染，满足理科教学需求。
- **验收标准：**
  1. 集成 KaTeX 渲染引擎
  2. 工具栏提供"插入公式"按钮，弹出公式编辑器
  3. 常用公式模板：分数、根号、积分、求和、矩阵、化学方程式
  4. 行内公式 `$...$` 和块级公式 `$$...$$`
  5. 实时预览
- **技术选型：** `katex` + `@tiptap/extension-mathematics` 或自建扩展
- **新依赖：** `katex`

### EDIT-RICH-04: 课程模板系统

- **描述：** 提供预设课程模板，教师可快速创建结构化课程。
- **验收标准：**
  1. 预设 4 种模板：讲授型、讨论型、实验型、复习型
  2. 每个模板预设时间线环节结构
  3. 教师可将当前课程"保存为模板"
  4. 从模板创建课程时复制结构 + 内容框架
  5. 模板存储在 `lesson_templates` 表
- **影响文件：** `packages/core/db/index.ts`、`server.ts` 新路由

### EDIT-RICH-05: AI 辅助内容生成增强

- **描述：** AI Agent 能够生成富文本格式的课程内容。
- **验收标准：**
  1. AI Agent 的 `lesson.create` 工具生成的 content 包含 HTML 结构
  2. 系统指令中引导 AI 使用语义化 HTML 标签
  3. AI 可建议相关图片资源（提供 Unsplash/Pexels 搜索建议）
  4. AI 可基于主题自动生成完整的结构化教学大纲

## Phase 24: 题库系统增强

### QUIZ-BANK-01: 题库管理

- **描述：** 建立独立于白板的题库系统，支持题目的 CRUD 和分类管理。
- **验收标准：**
  1. 新建 `question_bank` 表
  2. 题目管理界面（列表、搜索、过滤、分页）
  3. 导入导出（CSV/JSON）
  4. 题目标签 + 知识点标注
  5. 题目复用统计（被引用的测验数）
- **数据库：**
  ```sql
  CREATE TABLE question_bank (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    type TEXT NOT NULL,          -- single_choice, multi_choice, true_false, fill_blank, short_answer, code, matching
    content TEXT NOT NULL,       -- JSON: 题干、选项、答案等
    difficulty INTEGER DEFAULT 3, -- 1-5
    knowledge_points TEXT,       -- JSON 数组
    tags TEXT,                   -- JSON 数组
    subject TEXT,
    created_by TEXT NOT NULL,
    usage_count INTEGER DEFAULT 0,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
  );
  ```

### QUIZ-BANK-02: 多题型支持

- **描述：** 支持 7 种常见题型。
- **验收标准：**
  1. 单选题：N 选 1，选项文本 + 正确选项
  2. 多选题：N 选 M，部分得分可配置
  3. 判断题：正确/错误
  4. 填空题：文本输入，支持多空，支持模糊匹配
  5. 简答题：文本输入，教师手动评分或 AI 辅助评分
  6. 编程题：内嵌 Monaco Editor（代码编辑器），后端执行测试用例
  7. 拖拽匹配题：左右两列连线匹配
- **影响文件：** 前端组件（每种题型独立子组件）、后端评分逻辑

### QUIZ-BANK-03: 自动组卷

- **描述：** 教师设定规则，系统自动从题库抽题组成试卷。
- **验收标准：**
  1. 组卷表单：题目总数、各题型数量、难度分布、知识点覆盖
  2. AI 辅助：自然语言描述"生成一份高一函数章节中等难度的单元测试"
  3. 随机组卷（教学班每人不同卷，防止作弊）
  4. 试卷预览 + 手动替换题目
- **影响文件：** `packages/plugins/quiz-pro/index.ts` 扩展

### QUIZ-BANK-04: 限时考试模式

- **描述：** 实现正式的限时考试功能。
- **验收标准：**
  1. 考试配置：开始时间、结束时间、时长限制
  2. 倒计时显示（复用现有 CountdownTimer 组件）
  3. 时间到自动提交
  4. 防作弊选项：禁止切屏（Page Visibility API）、禁止复制粘贴、全屏模式
  5. 断线恢复：答案临时保存在 localStorage，重连后合并
  6. 考试状态追踪在 `exams` + `exam_scores` 表

### QUIZ-BANK-05: 测验结果分析

- **描述：** 为测验提供详细的分析统计。
- **验收标准：**
  1. 逐题正确率 + 选项分布图（选择题）
  2. 学生答题详情视图
  3. 知识点掌握度雷达图
  4. 班级成绩分布：平均分、中位数、标准差、及格率
  5. 学生端错题本

### QUIZ-BANK-06: 白板测验集成

- **描述：** 白板 quiz 元素与题库联动。
- **验收标准：**
  1. 白板 quiz 类型支持引用题库 ID
  2. 教师从题库弹窗中选择题目插入
  3. 课堂上测验结果关联到课程环节

## Phase 25: 学习路径与解锁机制

### PATH-LEARN-01: 课程体系结构

- **描述：** 建立课程层级结构。
- **验收标准：**
  1. 新建 `curriculum` 表（课程体系）和 `curriculum_nodes` 表（章/节/课）
  2. 教师端课程列表按树形结构展示
  3. 支持拖拽重排课程顺序和层级
  4. 学生端按体系浏览课程

### PATH-LEARN-02: 前置条件解锁

- **描述：** 激活现有的 progress_mode/progress_conditions 字段。
- **验收标准：**
  1. 教师可配置课程解锁规则
  2. 支持条件：完成前置课程、测验最低分、观看时长
  3. 学生端：未解锁课程灰色锁图标 + 解锁条件提示
  4. 解锁事件触发通知

### PATH-LEARN-03: 自适应学习推荐

- **描述：** AI 根据学习数据推荐个性化路径。
- **验收标准：**
  1. 学生首页"为你推荐"模块
  2. AI Agent 新增 `learning.recommend_next` action
  3. 基于：薄弱知识点、历史表现、学习速度
  4. 推荐内容：课程、补充练习、复习材料

### PATH-LEARN-04: 学习任务与截止日期

- **描述：** 建立任务管理系统。
- **验收标准：**
  1. 教师设置截止日期（关联课程/作业/测验）
  2. 学生待办清单视图
  3. 逾期标记 + 教师端逾期名单
  4. 新建 `student_tasks` 表追踪状态

## Phase 26: 学习进度仪表板

### DASH-STU-01: 学生端学习仪表板

- **描述：** 学生登录后的首页仪表板。
- **验收标准：**
  1. 总体进度环形图
  2. 最近学完/正在学的课程列表
  3. 待完成清单（按截止日期排序）
  4. 成绩趋势迷你图
  5. 与班级平均的对比（匿名）

### DASH-STU-02: 教师端教学监控面板

- **描述：** 教师一览全班学习状况。
- **验收标准：**
  1. 全班进度热力图
  2. 风险学生卡片（完成率<30% 或 分数<60）
  3. 课程完成漏斗
  4. 待批改作业数

### DASH-STU-03: 学习行为分析

- **描述：** 学习过程数据采集与分析。
- **验收标准：**
  1. 学习时长追踪（新建 `learning_sessions` 表）
  2. 活跃时段热力图
  3. 知识点覆盖矩阵
  4. 投入度评分（AI 综合评估）

### DASH-STU-04: 成绩预警与干预

- **描述：** 自动化成绩监控与干预机制。
- **验收标准：**
  1. 成绩趋势检测（连续下降自动预警）
  2. 教师端一键发送干预材料
  3. 预警历史追踪

## Phase 28: 实时授课基础

### LIVE-RTC-01: WebRTC 音视频直播
- **验收标准：**
  1. 教师端一键开播（摄像头 + 麦克风）
  2. 学生端观看直播流
  3. 教师可切换摄像头
  4. 延迟 < 500ms
- **技术选型：** WebRTC + 信令服务器（Socket.IO 扩展）

### LIVE-RTC-02: 屏幕/课件共享
- **验收标准：**
  1. 屏幕共享（Screen Capture API）
  2. 画中画（摄像头小窗叠加）
  3. 课件翻页同步

### LIVE-RTC-03: 互动控制
- **验收标准：**
  1. 举手功能
  2. 教师授权上麦
  3. 全体静音
  4. 在线状态指示

### LIVE-RTC-04: 录制与回放
- **验收标准：**
  1. 教师端录制按钮
  2. 录制内容：音视频 + 白板操作 + 课件翻页
  3. 存储为 WebM
  4. 课程关联回放列表
