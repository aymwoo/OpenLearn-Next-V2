# Architecture Decision: 插件系统能力增强 — 支撑数据密集型第三方插件

**状态：** 计划中
**日期：** 2026-06-24
**决策者：** Wu Xiangfeng

---

## 背景

v1.0-v3.0 的插件系统设计目标是为"工具型"插件服务——每个插件注册几个 command handler，在白板上创建交互元素。`quiz`、`rollcall`、`mindmap` 等内置插件都遵循此模式。

经过对考试插件的可行性分析，发现三个阻塞性缺口阻止了"数据密集型"第三方插件的开发：

1. 插件无法创建独立数据库表（只能用 KV 表 `plugin_storage`）
2. 前端 Extension Slot 不够用（仅 5 个，没有"独立管理面板"级别的插槽）
3. 插件无法复用主应用已有的 npm 依赖（recharts、jspdf 等）

此外还有两个增强性缺口：前端路由支持、文件下载 API。

---

## 决策

在不破坏现有插件系统架构的前提下，以最小增量方式补齐 5 项能力，使第三方开发者能够交付完整的考试/题库插件。

---

## 详细执行计划

### 改造 1：插件自主建表能力（P0，~50 行）

#### 现状

```typescript
// 插件 activate 中只能这样做：
const db = await ctx.resolve(IDatabaseToken);
// db.exec('CREATE TABLE ...') ← 理论上可行，但没有约定和隔离

// 实际都挤在 plugin_storage KV 表中：
db.prepare('INSERT INTO plugin_storage (plugin_id, key, value, updated_at) VALUES (?,?,?,?)')
  .run('ext-exam', 'question:q001', JSON.stringify({...}), Date.now());
```

数千道题目存在 KV 表中 → 每次查询全表扫描 → O(n) 性能。

#### 方案：命名空间隔离 + 幂等建表

在服务端 `PluginContext` 中增加 `db` 子对象：

```typescript
// packages/core/plugin-host/types.ts 中新增

interface PluginDatabaseAPI {
  /**
   * 确保插件专用表存在（幂等）。
   * 实际表名自动加前缀：plugin_{pluginId}_{tableName}
   * 
   * @example
   *   await ctx.db.ensureTable('questions', `
   *     id TEXT PRIMARY KEY,
   *     content TEXT NOT NULL,
   *     type TEXT NOT NULL DEFAULT 'single_choice',
   *     options TEXT,
   *     answer TEXT,
   *     difficulty INTEGER DEFAULT 3,
   *     tags TEXT,
   *     created_at INTEGER NOT NULL
   *   `);
   *   // 创建表: plugin_ext_exam_questions
   */
  ensureTable(tableName: string, schema: string): Promise<void>;

  /**
   * 获取带前缀的完整表名，用于手动 SQL 查询。
   * 始终使用此方法而非直接拼接表名。
   */
  table(tableName: string): string;

  /**
   * 删除插件创建的所有表（uninstall 时调用）。
   * 由 PluginHost 自动管理，插件通常不需要手动调用。
   */
  dropAllTables(): Promise<void>;
}
```

#### 上下文构建改动

```typescript
// packages/core/plugin-host/context-builder.ts

function buildPluginContext(pluginId: string, ...): PluginContext {
  const prefix = `plugin_${pluginId.replace(/[^a-zA-Z0-9_]/g, '_')}_`;

  const dbApi: PluginDatabaseAPI = {
    async ensureTable(tableName: string, schema: string) {
      const fullName = prefix + tableName;
      db.exec(`CREATE TABLE IF NOT EXISTS ${fullName} (${schema})`);
    },
    table(tableName: string) {
      return prefix + tableName;
    },
    async dropAllTables() {
      const tables = db.prepare(
        `SELECT name FROM sqlite_master WHERE type='table' AND name LIKE ?`
      ).all(prefix + '%') as { name: string }[];
      for (const t of tables) {
        db.exec(`DROP TABLE IF EXISTS ${t.name}`);
      }
    },
  };

  return {
    // ... existing services ...
    db: dbApi,  // NEW
  };
}
```

#### 卸载时自动清理

```typescript
// packages/core/plugin-host/index.ts — uninstallPlugin() 中增加：

async uninstallPlugin(pluginId: string): Promise<void> {
  // ... existing deactivate logic ...

  // 清理插件自建表
  const prefix = `plugin_${pluginId.replace(/[^a-zA-Z0-9_]/g, '_')}_`;
  const tables = this.db.prepare(
    `SELECT name FROM sqlite_master WHERE type='table' AND name LIKE ?`
  ).all(prefix + '%') as { name: string }[];
  for (const t of tables) {
    this.db.exec(`DROP TABLE IF EXISTS ${t.name}`);
  }

  // ... existing cleanup ...
}
```

#### 插件使用示例

```typescript
// ext-exam/server/index.js
export default {
  manifest: { id: 'ext-exam', name: '考试系统', version: '1.0.0' },
  activate: async (ctx) => {
    // 1. 建表（幂等）
    await ctx.db.ensureTable('questions', `
      id TEXT PRIMARY KEY,
      content TEXT NOT NULL,
      type TEXT NOT NULL,
      options TEXT,
      answer TEXT,
      difficulty INTEGER DEFAULT 3,
      knowledge_point TEXT,
      tags TEXT,
      created_at INTEGER NOT NULL,
      updated_at INTEGER NOT NULL
    `);

    await ctx.db.ensureTable('exam_sessions', `
      id TEXT PRIMARY KEY,
      title TEXT NOT NULL,
      question_ids TEXT NOT NULL,
      duration_minutes INTEGER NOT NULL,
      class_id TEXT NOT NULL,
      start_time INTEGER,
      end_time INTEGER,
      status TEXT DEFAULT 'draft',
      created_at INTEGER NOT NULL
    `);

    await ctx.db.ensureTable('exam_answers', `
      id TEXT PRIMARY KEY,
      session_id TEXT NOT NULL,
      student_id TEXT NOT NULL,
      question_id TEXT NOT NULL,
      answer TEXT,
      score REAL,
      submitted_at INTEGER,
      UNIQUE(session_id, student_id, question_id)
    `);

    // 2. 注册命令处理器
    const qt = ctx.db.table('questions');  // → 'plugin_ext_exam_questions'

    ctx.commandBus.registerHandler('exam.question.create', {
      execute: async (cmd) => {
        const p = cmd.payload;
        const id = uuidv7();
        ctx.db.run(
          `INSERT INTO ${qt} (id, content, type, options, answer, difficulty, knowledge_point, tags, created_at, updated_at) VALUES (?,?,?,?,?,?,?,?,?,?)`,
          id, p.content, p.type, JSON.stringify(p.options), p.answer, p.difficulty || 3, p.knowledgePoint || '', JSON.stringify(p.tags || []), Date.now(), Date.now()
        );
        return { id };
      }
    });
  }
};
```

### 改造 2：扩展前端 Extension Slot（P0，~100 行）

#### 现状

```typescript
// src/plugin-host/types.ts — 仅 5 个 slot
export type ExtensionSlot =
  | 'teacher.tab'
  | 'student.view'
  | 'classroom.tool'
  | 'teacher.dashboard.widget'
  | 'student.lesson.tool';
```

考试插件需要一个完整的教师端管理面板来承载"题库管理 → 考试配置 → 成绩分析"的工作流。现有 slot 都不合适。

#### 方案：新增 3 个 slot + 支持配置项

```typescript
// src/plugin-host/types.ts

export type ExtensionSlot =
  // 现有 (保持不变)
  | 'teacher.tab'              // 教师顶部 Tab（已有，但未在 UI 中使用）
  | 'student.view'             // 学生视图
  | 'classroom.tool'           // 课堂工具箱按钮
  | 'teacher.dashboard.widget' // 教师仪表板小组件
  | 'student.lesson.tool'      // 学生课堂工具
  // 新增
  | 'teacher.panel'            // 教师独立全宽管理面板（NEW）
  | 'student.fullscreen'       // 学生全屏视图，用于考试模式（NEW）
  | 'global.setting'           // 全局设置页扩展（NEW）
  ;

export interface ExtensionPointConfig {
  id: string;
  label: string;
  icon?: string;               // Lucide 图标名称
  component: () => Promise<{ default: React.ComponentType<any> }>;
  position?: number;            // 排序优先级（越小越靠前）
  pluginId: string;
  // 新增字段
  route?: string;               // 可选子路由，如 '/exam/questions'（见改造 4）
  slotProps?: Record<string, any>; // 传递给组件的额外 props
}
```

#### 在 App.tsx 中渲染新 slot

```tsx
// 教师面板 — 在教师视图的 Tab 栏中渲染
{teacherTab === 'plugins' && (
  <div className="flex-1 overflow-auto">
    <ExtensionPointRenderer slot="teacher.panel" />
  </div>
)}

// 学生全屏 — 在特定条件（如考试模式）下使用
{studentView === 'fullscreen-plugin' && (
  <ExtensionPointRenderer slot="student.fullscreen" />
)}
```

#### 向后兼容

现有插件不受影响。新增 slot 仅对新插件可见。`ExtensionPointRenderer` 对未注册的 slot 返回 null。

### 改造 3：插件共享依赖机制（P0，~80 行）

#### 现状

插件是独立的 JS/TS 字符串，运行时通过 `Blob URL + import()` 或 Worker 加载。插件内部 `import` 只能解析自己的代码，无法引用 `recharts`、`jspdf` 等主应用已安装的依赖。

考试插件的成绩分析需要图表，导出需要 Excel/PDF——这两个功能要么自带库（导致插件包体积爆炸），要么无法实现。

#### 方案：require() 白名单机制

##### 3.1 定义白名单

```typescript
// packages/core/plugin-host/types.ts

/** 插件可引用的主应用共享模块白名单 */
export const PLUGIN_SHARED_MODULES = [
  'recharts',          // 图表（成绩分布、趋势）
  'react-markdown',    // Markdown 渲染
  'jspdf',             // PDF 导出
  'jspdf-autotable',   // PDF 表格
  'xlsx',              // Excel 导出
  'konva',             // Canvas 2D（已共享）
  'react-konva',       // Konva React 绑定
  'react-konva-utils', // Konva 工具
  'lucide-react',      // 图标
  'uuid',              // ID 生成
] as const;

// 在主应用 module cache 中注册
import * as recharts from 'recharts';
import * as jspdf from 'jspdf';
// ...

// PluginContext 中暴露
ctx.require('recharts')  // → recharts 模块
ctx.require('jspdf')     // → jspdf 模块
```

##### 3.2 上下文构建

```typescript
// packages/core/plugin-host/context-builder.ts

import * as Recharts from 'recharts';
import * as JsPDF from 'jspdf';
import * as JsPDFAutoTable from 'jspdf-autotable';
import * as XLSX from 'xlsx';
// ... (仅服务端需要的模块)

const SHARED_MODULES: Record<string, any> = {
  recharts: Recharts,
  jspdf: JsPDF,
  'jspdf-autotable': JsPDFAutoTable,
  xlsx: XLSX,
  konva: Konva,
  'react-konva': ReactKonva,
  'react-konva-utils': ReactKonvaUtils,
  'lucide-react': LucideReact,
  uuid: UUID,
};

function buildPluginContext(...): PluginContext {
  return {
    // ... existing ...
    require: (moduleName: string) => {
      if (!PLUGIN_SHARED_MODULES.includes(moduleName as any)) {
        throw new Error(`Plugin cannot require "${moduleName}". Allowed: ${PLUGIN_SHARED_MODULES.join(', ')}`);
      }
      return SHARED_MODULES[moduleName];
    },
  };
}
```

##### 3.3 前端同理

```typescript
// src/plugin-host/plugin-host.ts — FrontendPluginContext 中

const FRONTEND_SHARED_MODULES: Record<string, any> = {};
// 在 initialize 时填充
export async function loadSharedModules() {
  FRONTEND_SHARED_MODULES['recharts'] = await import('recharts');
  FRONTEND_SHARED_MODULES['lucide-react'] = await import('lucide-react');
  // ... 按需加载
}
```

##### 3.4 插件打包约定

插件开发者应将这些模块标记为 external，由运行时注入：

```javascript
// ext-exam/rollup.config.js
export default {
  external: ['recharts', 'jspdf', 'jspdf-autotable', 'xlsx', 'lucide-react'],
  // 这些模块不打包进插件，运行时由 ctx.require() 提供
};
```

##### 3.5 安全性

- 严格白名单：只有 `PLUGIN_SHARED_MODULES` 中的模块可被 require
- Worker 模式：共享模块通过 `postMessage` 序列化传递（仅数据，不能传递函数引用）
- 审计日志：记录插件 require 调用

### 改造 4：前端路由支持（P1，~40 行）

#### 现状

插件组件是静态挂载到 slot 的，无法根据 URL 切换内部视图。

#### 方案：在 ExtensionPointRenderer 中注入 route prop

```typescript
// src/plugin-host/extension-point-renderer.tsx

export function ExtensionPointRenderer({ slot, route }: ExtensionPointRendererProps) {
  const host = usePluginHost();
  const extensions = host.getExtensions(slot);

  if (extensions.length === 0) return null;

  return (
    <>
      {extensions.map((ext) => (
        <ExtensionErrorBoundary key={ext.id} fallback={...}>
          <Suspense fallback={<LoadingSkeleton />}>
            {React.createElement(
              React.lazy(ext.component),
              { route: ext.route || route }  // ← 注入当前路由
            )}
          </Suspense>
        </ExtensionErrorBoundary>
      ))}
    </>
  );
}
```

#### 插件端使用

```tsx
// ext-exam/frontend/ExamManager.tsx
export default function ExamManager({ route }: { route?: string }) {
  // 简单的 hash 路由
  const [hash, setHash] = useState(window.location.hash || '#questions');

  useEffect(() => {
    const onHashChange = () => setHash(window.location.hash);
    window.addEventListener('hashchange', onHashChange);
    return () => window.removeEventListener('hashchange', onHashChange);
  }, []);

  switch (hash) {
    case '#questions': return <QuestionBank />;
    case '#exams':     return <ExamList />;
    case '#results':   return <ScoreDashboard />;
    default:           return <QuestionBank />;
  }
}
```

### 改造 5：文件下载 API（P1，~20 行）

#### 现状

插件可以将数据发送到后端生成 CSV/Excel，但无法触发浏览器下载。

#### 方案：UI Service 增加下载方法

```typescript
// src/plugin-host/types.ts — IUIService 接口增加

export interface IUIService {
  showToast(title: string, message: string, type: 'info' | 'success' | 'warning'): void;
  showModal(title: string, content: React.ReactNode): void;
  closeModal(): void;
  // NEW
  downloadFile(data: Blob | string, filename: string, mimeType?: string): void;
}
```

#### 实现

```typescript
// src/services/ui-service.ts

downloadFile(data: Blob | string, filename: string, mimeType = 'application/octet-stream'): void {
  const blob = typeof data === 'string'
    ? new Blob([data], { type: mimeType })
    : data;
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
}
```

## 风险

| 风险 | 缓解 |
|------|------|
| `ensureTable` 不加限制可能被恶意插件创建大量表 | 限制每个插件最多 10 张表，卸载时自动清理 |
| `ctx.require()` 在 Worker 模式下的序列化限制 | Worker 插件仅允许 require 纯数据模块（recharts 不可用），文档说明 |
| 新增 slot 在 App.tsx 中的渲染位置选择 | 先渲染 `teacher.panel` 在教师 Tab 栏中，后续可调整 |
