# Architecture Decision: 微前端模块内聚 + EventBus 集成

**状态：** 计划中
**日期：** 2026-06-24
**决策者：** Wu Xiangfeng

---

## 背景

OpenLearnV2 v2.0 引入了 Module Federation 微前端，将白板（3380 行）和课件（85 行）拆分为独立子应用。经过代码审查，两个子应用对 MfeContext 的实际运行时依赖非常有限。

### 当前架构三大问题

1. **React 多实例冲突**：`dispatcher.getOwner is not a function` 频繁出现
2. **运维复杂度**：3 个构建/服务进程
3. **架构不统一**：白板直接裸操作 Socket.IO，30+ 处 `type: 'refresh'` 事件全部绕过 EventBus 和审计日志

---

## 决策

1. **合并源码**：mfe-whiteboard、mfe-courseware 移入 `src/features/`，作为主应用本地模块
2. **删除 MF**：移除 Module Federation 基础设施（`src/mfe/`、`@module-federation/*`）
3. **混合事件策略**：
   - **高频实时事件**（temp-draw、temp-end、segment-change）→ 保留直连 Socket，不经 EventBus
   - **结构性操作事件**（element_added、deleted、board_cleared 等）→ 客户端发布到前端 EventBus，经 SocketBridge 到达服务端 EventBus，自动写入审计日志
4. **直接导入服务**：`import { socketService } from '@/services/socket-service'` + `import { useAppStore } from '@/store/appStore'`，不使用 DI 代理

## 通信路径对比

### 改前
```
Client A 白板
  socketService.emit('whiteboard-update', { roomId, type: 'refresh' })
    → Server Socket.IO handler
    → io.to(roomId).emit('whiteboard-sync', { type: 'refresh' })
    → Client B 白板 socketService.on('whiteboard-sync') → 重新拉取 elements
         ❌ 没有审计日志
         ❌ AI Agent 无法订阅
```

### 改后（结构性操作）
```
Client A 白板
  eventBus.publish({ type: 'whiteboard.element_added', payload: { lessonId, elementId } })
    → Frontend EventBus (in-memory)
    → SocketBridge → Socket.IO emit('server:whiteboard.event', payload)
    → Server Socket.IO handler
    → server-side eventBus.publish({ type: 'whiteboard.element_added', ... })
        ├─ → SQLite events 表（审计日志 ✅）
        └─ → io.to(roomId).emit('whiteboard-sync', payload) → Client B 重新拉取
```

### 改后（实时绘制，不变）
```
Client A 白板
  socketService.emit('whiteboard-update', { roomId, type: 'temp-draw', payload })
    → Server Socket.IO handler
    → io.to(roomId).emit('whiteboard-sync', { type: 'temp-draw', payload })
    → Client B 白板渲染中间笔画
         ❌ 仍然没有审计日志（这是故意的——高频事件不写入DB）
```

### Step 1：创建前端 EventBus Bridge（30min）

当前前端没有独立可导入的 EventBus 实例。`MfeEventBusWrapper` 是 MFE 专用的包装器，且依赖于 SocketBridge。我们需要一个更简单的、可供白板直接使用的前端 EventBus。

#### S1.1 创建 `src/services/event-bus.ts`
```typescript
// 轻量级的前端 EventBus，支持 publish/subscribe
// 与 MfeEventBusWrapper 功能等价，但不绑定特定 MFE name

import type { PlatformEvent } from '../../packages/core/event-bus';

type EventHandler = (event: PlatformEvent) => void;

class FrontendEventBus {
  private handlers = new Map<string, Set<EventHandler>>();

  subscribe(eventType: string, handler: EventHandler): () => void {
    if (!this.handlers.has(eventType)) {
      this.handlers.set(eventType, new Set());
    }
    this.handlers.get(eventType)!.add(handler);
    return () => this.handlers.get(eventType)?.delete(handler);
  }

  async publish(event: PlatformEvent): Promise<void> {
    const handlers = this.handlers.get(event.type);
    if (handlers) {
      for (const handler of handlers) {
        try { handler(event); } catch (e) { console.error('[EventBus] handler error:', e); }
      }
    }
    // 通过 SocketBridge 转发到服务端
    if (event.type.startsWith('whiteboard.')) {
      this.socketEmit?.(event);
    }
  }

  // 由外部注入 Socket 转发逻辑
  private socketEmit?: (event: PlatformEvent) => void;
  setSocketBridge(fn: (event: PlatformEvent) => void) {
    this.socketEmit = fn;
  }
}

export const frontendEventBus = new FrontendEventBus();
```

#### S1.2 在 `main.tsx` 中初始化
```typescript
// src/main.tsx
import { frontendEventBus } from './services/event-bus';
import { socketService } from './services/socket-service';
import { v7 as uuidv7 } from 'uuid';

// 将白板事件通过 Socket.IO 转发到服务端

  socketService.emit('whiteboard-event', {
    type: event.type,
    payload: event.payload,
    id: event.id,
    timestamp: event.timestamp,
  });
});
```

```typescript
// 根据操作上下文确定事件类型
function getWhiteboardEventType(context: {
  action: 'add' | 'update' | 'delete' | 'clear' | 'quiz' | 'rollcall' | 'slide' | 'annotate' | 'draw' | 'ai';
  elementType?: string;
}): string {
  switch (context.action) {
    case 'add': return 'whiteboard.element_added';
    case 'update': return 'whiteboard.element_updated';
    case 'delete': return 'whiteboard.element_deleted';
    case 'clear': return 'whiteboard.board_cleared';
    case 'quiz': return 'whiteboard.quiz_submitted';
    case 'rollcall': return 'whiteboard.rollcall_picked';
    case 'slide': return 'whiteboard.slide_changed';
    case 'annotate': return 'whiteboard.annotation_completed';
    case 'draw': return 'whiteboard.drawing_committed';
    case 'ai': return 'whiteboard.ai_generated';
    default: return 'whiteboard.element_updated';
  }
}
```

### Step 4：服务端增强 — 白板事件接入审计（30min）

#### S4.1 在 server.ts 中添加白板事件处理

```typescript
// 新增 Socket.IO 事件处理：接收来自前端的白板结构化事件
// 发布到服务端 EventBus（自动写入 events 表 + 广播给其他客户端）

socket.on('connection', (socket) => {
  // ... 现有处理 ...

  socket.on('whiteboard-event', (data: {
    type: string;
    payload: { lessonId: string; elementId?: string; elementType?: string; segmentId?: string };
    id: string;
    timestamp: number;
  }) => {
    // 1. 发布到服务端 EventBus（自动审计日志）
    kernelContainer.eventBus.publish({
      id: data.id,
      type: data.type,          // 例如 'whiteboard.element_added'
      source: 'whiteboard',
      payload: data.payload,
      timestamp: data.timestamp,
      correlationId: data.payload.lessonId,
    });

    // 2. 广播到同课程房间的其他客户端（触发重新拉取）
    const lessonId = data.payload?.lessonId;
    if (lessonId) {
      socket.to(`lesson-${lessonId}`).emit('whiteboard-sync', {
        type: 'refresh',
        sourceEvent: data.type,
        payload: data.payload,
      });
    }
  });
});
```

#### S4.2 效果

从此，每次白板元素的新增/修改/删除/清空操作，都会自动写入 `events` 表：
```sql
SELECT type, source, datetime(timestamp/1000, 'unixepoch') as time, payload
FROM events
WHERE type LIKE 'whiteboard.%'
ORDER BY timestamp DESC
LIMIT 20;
```
