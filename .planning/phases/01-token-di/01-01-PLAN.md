实现 packages/core/di/errors.ts — 具名错误类层次结构（per D-07）：
- TokenError(tokenName, message)：Token 自身错误（命名格式验证失败时使用）
- DuplicateRegistrationError(tokenName)：重复注册错误。错误信息包含 tokenName 和 "Use registerOrReplace() to overwrite" 提示（per D-08）
- MissingDependencyError(tokenName, missingDeps：string[])：注册时缺失依赖。错误信息列出缺失的 deps
- CircularDependencyError(cycleTokens：string[])：循环依赖错误。错误信息列出参与循环的 Token（per ROADMAP SC-4）
- HasDependentError(tokenName, dependents：string[])：注销被阻止。错误信息列出依赖方和 "Unregister them first" 提示（per D-09）
- 所有错误类设置 this.name 为类名（如 'TokenError'），使 `instanceof` 和调试可用
- 错误信息前缀使用 `[Token]` 或 `[ServiceRegistry]` 标签，遵循项目 console 日志惯例（如 `[CommandBus]`）

实现 packages/core/di/token.ts — Token<T> 泛型类（per D-01, D-04）：
- 使用 phantom 类型参数：`private _phantomService!: T`（仅用于类型推导，不在运行时使用）
- 构造函数接收 name：string 参数，存储为 `public readonly name`
- 构造函数内验证命名格式：正则 `/^@[a-zA-Z0-9_-]+\/[a-zA-Z0-9_-]+:[a-zA-Z0-9_]+$/`（per RESEARCH.md Pattern 1 和 Pitfall 4）
  格式不匹配时抛出 TokenError
- 构造函数内验证 name 非空：空字符串或非 string 类型抛出 TokenError
- 不需要 version 字段（per D-15）
- 从 errors.ts 导入 TokenError（使用 .js 扩展名 ESM 导入）

实现 packages/core/di/__tests__/token.test.ts — Token 单元测试：
- 使用 vitest 的 describe/it/expect
- 测试用例（per D-16）：
  a) 创建 Token 实例，验证 name 属性等于传入的字符串
  b) 验证泛型类型推导：`new Token<ICommandBusService>('@openlearn/core:ICommandBusService')` 编译通过（类型级测试，TypeScript 编译即证明）
  c) 空字符串 name 抛出 TokenError
  d) 不符合命名格式的 name（如 'no-scope'、'@scope/domain:Name With Space'、'@scope/domain:中文名'）抛出 TokenError
  e) 合法格式通过：`@openlearn/core:ICommandBusService`、`@openlearn/plugin:IQuizGenerator`
  f) 不同 name 的两个 Token 实例不相等（标识符唯一性验证）
- 测试文件导入路径：从 `../token.js` 和 `../errors.js`（ESM 规范）
