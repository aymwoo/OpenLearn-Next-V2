    This is the most complex component in the phase. Implementation requirements:

    1. **Props interface:**
    ```typescript
    interface MfeLoaderCoreProps {
      name: string;                     // Remote module name (e.g., 'mfe_whiteboard')
      entry?: string;                   // Pre-resolved entry URL (skip API lookup)
      props?: Record<string, any>;      // Props to pass to remote mount()
      timeout?: number;                 // Load timeout in ms (default from config or 30000)
      loadingFallback?: React.ComponentType;  // Per-instance override (D-03)
      errorFallback?: React.ComponentType<MfeErrorFallbackProps>;  // Per-instance override (D-03)
      mfeRef?: React.Ref<MfeControllerRef>;   // Explicit controller ref (D-19)
      onError?: (error: Error) => void;       // Error callback for parent
      onLoad?: () => void;                   // Success callback for parent
    }
    ```

    2. **State machine:** Three states: `'loading' | 'loaded' | 'error'`. Track via `useState`.

    3. **On mount (useEffect with [name, entry, props] deps):**
      - Set state to 'loading'.
      - Determine entry URL: if `entry` prop provided, use it directly. Otherwise, use `fetchRemoteEntry(name)` from `api.ts` with cache (D-23, D-24).
      - Call `loadRemote<{ default: any; createMfeApp?: Function }>(`${name}/App`)` from `@module-federation/runtime`.
      - Check export format (D-12):
        - If `mod.createMfeApp`: call `const app = mod.createMfeApp(mfeContext)` — single init (D-08).
        - If `mod.default` (React component): wrap via `wrapReactComponent(mod.default)` (D-12 backward compat).
      - Call `app.mount(container, props)` (D-06, D-09 — both can be async).
      - Store mount return value (MfeAppInstance) in ref for unmount/update.
      - Handle styles (D-10): if `app.styles` array present, inject `<link>` elements into document.head with IDs like `mfe-style-${name}-${index}`. Track injected elements for cleanup.
      - Set state to 'loaded'.

    4. **On error (catch):**
      - Set state to 'error'.
      - Store error for fallback component.
      - Call `onError?.(error)` if provided.

    5. **Timeout (D-18):**
      - Start a timeout timer on mount (default 30s, overridden by props.timeout or config).
      - If timeout fires before load completes, reject with error message including "timeout".
      - Clear timeout on unmount or successful load.

    6. **Cleanup (useEffect return) (D-19, D-21, D-22):**
      - Initiate dual-path unmount:
        a. Call `mfeAppInstance.unmount()` (the remote's cleanup).
        b. Start a 5-second forced timeout (D-22).
        c. `Promise.race([mfeAppInstance.unmount(), timeoutPromise])` — if remote unmount() takes >5s, force cleanup.
      - On unmount completion (or timeout):
        - Call `root.unmount()` (React 19 root destruction, D-04, D-21).
        - Remove injected `<link>`/`<style>` elements from document.head (D-10, D-21).
        - Run leak detector check (D-20).
        - Clear all refs.

    7. **Rendering:**
      - Returns a `<div ref={containerRef}>` element where the remote mounts.
      - On error state: render null (MfeLoader.tsx wraps in ErrorBoundary that handles error display).
      - On loading state: render `loadingFallback` component (from props, or from config, or default MfeLoadingFallback).

    8. **Backward Compat Wrapper (D-12):**
    ```typescript
    function wrapReactComponent(Component: React.ComponentType<any>) {
      return {
        mount: async (container: HTMLElement, props?: Record<string, any>) => {
          const root = createRoot(container);
          root.render(React.createElement(Component, props));
          return {
            unmount: async () => { root.unmount(); },
            update: async (newProps: Record<string, any>) => {
              root.render(React.createElement(Component, newProps));
            },
          };
        },
        styles: [],
      } satisfies MfeAppLifecycle;
    }
    ```

    9. **Leak detector integration (D-20):**
      - Call `createLeakDetector()` in cleanup path, run `.check()` after unmount.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05 | Tampering | loadRemote() remote module injection | mitigate | Remote modules only loaded from dev servers (controlled ports 5174/5175) or SQLite-registered URLs (D-23). MfeLoader does NOT accept arbitrary URL strings — resolves via registered name. |
| T-11-06 | Denial of Service | Remote component render crash | mitigate | Per-instance ErrorBoundary (D-14) catches render-phase crashes. One remote crashing does not affect other remotes or the host shell. |
| T-11-07 | Denial of Service | Unmount hang | mitigate | 5-second forced timeout (D-22) via Promise.race ensures cleanup completes even if remote unmount() hangs indefinitely. |
| T-11-08 | Information Disclosure | Error stack traces shown to users | mitigate | ErrorBoundary catches errors but the default MfeErrorFallback shows only generic messages (extensionLoadError i18n key). Custom fallback may override but is controlled by host code. |
| T-11-09 | Tampering | Style injection from remote styles array | mitigate | Remote's styles array controlled by SQLite-registered remotes (first-party only in this phase). In production, style URLs are validated at registration time. For unverified third-party remotes, styles are reviewed as part of plugin approval. |
| T-11-SC | Tampering | npm packages | mitigate | No new packages. All dependencies (@module-federation/runtime, react, lucide-react) already installed and verified per RESEARCH.md. |
