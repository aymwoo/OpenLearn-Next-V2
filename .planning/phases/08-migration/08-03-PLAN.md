# Plan: 08-03 — AI Plugins Migration & Legacy Cleanup

**Objective:** 重构应用级的 AI Planner 和 AI Submit Injector 插件，实现 `PluginHost.togglePlugin()` 并在 ServiceRegistry 中注册 `IPluginHostToken`，最后物理删除旧版的 `plugin-runtime` 代码及在内核与路由层面的全部遗留引用。

## Tasks

### Task 1: 声明 IPluginHostToken 并实现 PluginHost.togglePlugin()

**Files:**
- `packages/core/di/interfaces.ts` — 导出 `IPluginHostToken`
- `packages/core/plugin-host/index.ts` — 实现 `togglePlugin(pluginId)` 方法
- `packages/core/kernel/index.ts` — 注册 `IPluginHostToken` 到 ServiceRegistry

**Description:**
- 在 `interfaces.ts` 中声明 `IPluginHostToken`：
  ```typescript
  export const IPluginHostToken = new Token<PluginHost>(
    '@openlearn/core:IPluginHost'
  );
  ```
- 在 `PluginHost` 中实现 `togglePlugin(pluginId: string): Promise<string>`。根据当前状态激活或停用（处理 inline/worker 模式），并更新 DB 中该插件的 `status` 为 `active` 或 `disabled`。
- 在 `Kernel` 的 constructor 中将 `this.pluginHost` 注册至 `serviceRegistry`。

**Verification:**
- `PluginHost` 编译无错，能正常完成插件激活与停用状态的切换。

### Task 4: 物理删除 Legacy PluginRuntime 与清理内核/服务引用

**Files:**
- `packages/core/plugin-runtime/index.ts` — 物理删除此文件与文件夹。
- `packages/core/kernel/index.ts` — 移除 `PluginRuntime` 引用和实例化代码，将 AI 插件加入 Kernel 自动加载的系统插件列表（激活失败为 Soft fail）。
- `server.ts` — 移除 `pluginRuntime` 相关调用，将其替换为 `pluginHost`。修改 `/api/plugins` 相关 HTTP 接口。

**Description:**
- 物理删除 `packages/core/plugin-runtime/`。
- 修改 `packages/core/kernel/index.ts`：
  - 移除对 `PluginRuntime` 的 import。
  - 删除 `public readonly pluginRuntime: PluginRuntime;` 属性声明与 constructor 内的实例化。
  - 更新 `bootstrapSystemPlugins()` 列表，引入并注册 `AiPlannerPlugin` 与 `AiSubmitInjectorPlugin`。
  - 对 AI 插件的加载与激活异常进行 `try/catch` 捕获并 `console.warn`，执行 Soft fail，不阻碍内核启动。
- 修改 `server.ts`：
  - 启动阶段不再调用 `kernelContainer.pluginRuntime.loadFromDB()`，而是由内核在启动时自动调用 `bootstrapSystemPlugins()`。
  - 修改 `app.get('/api/plugins')` 等路由以直接调用 `kernelContainer.pluginHost.listPlugins()`。
  - 修改 `/api/plugins/:id/toggle` 路由，调用 `plugin.toggle` 指令或直接调用 `kernelContainer.pluginHost.togglePlugin(id)`。
  - 修改 `/api/plugins/:id` DELETE 路由，调用 `plugin.uninstall` 指令或直接调用 `kernelContainer.pluginHost.uninstallPlugin(id)`。

**Verification:**
- 运行 `npx vitest run packages/core/__tests__/legacy-cleanup.test.ts` 验证 `pluginRuntime` 被成功移除，且服务器接口依然工作正常。
- 启动服务器，通过浏览器/接口访问插件列表与操作，一切工作正常。
