# Plan: 08-02 — Business Built-in Plugins Migration

**Objective:** 重构业务逻辑层面的内置插件（管理插件 `management.ts` 和教学/白板插件 `builtin.ts`）至 ESM + Token DI 模式，从 `server.ts` 中完全移除这些插件的硬编码加载，统一交给 Kernel 在启动时通过 `PluginHost` 加载与激活。

## Tasks

### Task 1: 重构 ManagementPlugin (教务管理)

**Files:**
- `packages/plugins/management.ts` — 重构为标准 ESM 导出的 `ManagementPlugin` 对象。

**Description:**
- 剥离对 `kernelContainer` 的直接引用。
- 声明并导出 `ManagementPlugin`：
  ```typescript
  export const ManagementPlugin = {
    manifest: {
      id: '@openlearn/plugin-management',
      name: 'LMS Management Plugin',
      version: '1.0.0',
      requires: [
        '@openlearn/core:ICommandBusService@^1.0.0',
        '@openlearn/core:IActionRegistryService@^1.0.0',
        '@openlearn/core:IDatabase@^1.0.0',
      ],
      capabilitiesProposed: ['class:read', 'class:write', 'student:read', 'student:write'],
    },
    activate: async (ctx) => {
      const commandBus = ctx.services.resolve(ICommandBusServiceToken);
      const actionRegistry = ctx.services.resolve(IActionRegistryServiceToken);
      const db = ctx.services.resolve(IDatabaseToken);

      // 注册 class.create, student.create, class.add_student, class.get_students,
      // schedule.create, attendance.record, assignment.create, assignment.submit, assignment.grade 等指令处理器
    },
    deactivate: async () => { /* 清理 Handler */ }
  };
  ```

**Verification:**
- 运行 `npx vitest run packages/plugins/__tests__/management.test.ts` 进行验证。

### Task 3: Kernel 启动集成与 server.ts 清理

**Files:**
- `packages/core/kernel/index.ts` — 将 `ManagementPlugin` 和 `BuiltinPlugin` 加入 Kernel 自动加载的系统插件列表，并在激活失败时抛出错误（Hard crash）。
- `server.ts` — 移除对旧的 `bootstrapVFSPlugins`, `bootstrapProcessPlugins`, `bootstrapManagementPlugins`, `bootstrapBuiltinPlugins` 的 import 和直接调用。

**Description:**
- 在 `packages/core/kernel/index.ts` 中引入 `ManagementPlugin` 和 `BuiltinPlugin`：
  ```typescript
  import { ManagementPlugin } from '../../plugins/management.js';
  import { BuiltinPlugin } from '../../plugins/builtin.js';
  ```
- 更新 Kernel 的 `bootstrapSystemPlugins()` 列表，增加这两个插件。
- 确保系统启动时在 `plugins` 表中检查并插入记录（如果不存在），将其设置为 `execution_mode = 'inline'`。
- 在 `server.ts` 中删除以下导入和调用：
  ```typescript
  import { bootstrapBuiltinPlugins } from './packages/plugins/builtin.js';
  import { bootstrapVFSPlugins } from './packages/plugins/vfs.js';
  import { bootstrapProcessPlugins } from './packages/plugins/process.js';
  import { bootstrapManagementPlugins } from './packages/plugins/management.js';
  ```
  以及它们在 `server.ts` 启动时的直接调用。

**Verification:**
- 编译并启动服务器，没有旧的 bootstrap 函数调用错误，且这四个插件自动通过 Kernel 完成了加载。
- 运行全量单元测试没有 Regression。

## Verification Criteria

1. `ManagementPlugin` 彻底剥离全局 kernelContainer 引用，且激活失败导致 Hard crash。

3. `server.ts` 中的 4 个 legacy 插件 bootstrap 导入和调用被物理删除。
4. Kernel 初始化时自动注册并激活这两个业务插件，并在 `plugins` 表中写入对应的系统插件记录。
5. 所有新增的 2 个单元测试通过，全量测试全部通过。
