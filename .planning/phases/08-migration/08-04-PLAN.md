# Plan: 08-04 — Third-Party Plugins Packaging & Worker Sandbox Testing

**Objective:** 为两个第三方插件（智能随堂测验 Quiz 和 随机点名抽问 Roll Call）建立 TS 源码目录，编写 `build-plugins.mjs` 构建脚本打包输出 ZIP 至 `dist/plugins/`，在 Kernel 中实现 Worker 模式 Seeding，并编写跨边界 RPC 和 Event 转发的端到端集成测试。

## Tasks

### Task 1: 建立 Quiz 与 Rollcall 插件的 TS 源码和清单

**Files:**
- `packages/plugins/quiz/manifest.json` — Quiz 插件清单
- `packages/plugins/quiz/index.ts` — Quiz 插件入口代码
- `packages/plugins/rollcall/manifest.json` — Roll Call 插件清单
- `packages/plugins/rollcall/index.ts` — Roll Call 插件入口代码

**Description:**
- 在 `packages/plugins/quiz/manifest.json`：
  ```json
  {
    "id": "ext-quiz-generator",
    "name": "Quiz Component Plugin",
    "version": "1.0.0",
    "main": "index.js",
    "requires": [
      "@openlearn/core:ICommandBusService@^1.0.0",
      "@openlearn/core:IActionRegistryService@^1.0.0"
    ],
    "capabilitiesProposed": ["quiz:write"]
  }
  ```
- 在 `packages/plugins/quiz/index.ts` 中，使用 standard ESM plugin 模式编写随堂测验逻辑：
  ```typescript
  import { ICommandBusServiceToken, IActionRegistryServiceToken } from '@openlearn/core'; // 或是相对路径

  export default {
    activate: async (ctx) => {
      const commandBus = ctx.services.resolve(ICommandBusServiceToken);
      const actionRegistry = ctx.services.resolve(IActionRegistryServiceToken);

      await actionRegistry.register({
        id: 'ext-quiz-create',
        commandType: 'quiz.create',
        description: 'Create a multiple-choice quiz on the whiteboard for a lesson',
        capabilityRequired: 'whiteboard:write',
        inputSchema: { /* schema */ }
      });

      await commandBus.registerHandler('quiz.create', {
        execute: async (command) => {
          // 调用 whiteboard.draw 命令
        }
      });
    }
  };
  ```
- 在 `packages/plugins/rollcall/manifest.json` 和 `index.ts` 中实现随机学生点名逻辑，注册 `rollcall.pick` 命令。

**Verification:**
- 代码可编译，且 Manifest 中版本和依赖 Token 兼容。

### Task 2: 实现 scripts/build-plugins.mjs 打包构建脚本

**Files:**
- `scripts/build-plugins.mjs` — 打包脚本
- `package.json` — 更新 build 指令，加上构建打包步骤
- `.gitignore` — 忽略 `dist/plugins/` 目录

**Description:**
- 编写 `scripts/build-plugins.mjs`。使用 `esbuild` 编译 `packages/plugins/quiz/index.ts` 和 `packages/plugins/rollcall/index.ts` 到单文件，再配合 `jszip` 将对应的 `manifest.json` 和编译后的 `index.js` 内存压缩并写入到 `dist/plugins/ext-quiz-generator.zip` 和 `dist/plugins/ext-roll-call.zip`。
- 修改 `package.json` 的 `build` script：
  ```json
  "build": "node scripts/build-plugins.mjs && vite build && esbuild server.ts --bundle --platform=node --format=cjs --packages=external --sourcemap --outfile=dist/server.cjs"
  ```
- 修改 `.gitignore` 加上 `dist/plugins/`。

**Verification:**
- 运行 `node scripts/build-plugins.mjs` 可以在 `dist/plugins/` 生成两个 `.zip` 压缩文件，且能成功解压出合法的 `manifest.json` 和 `index.js`。

### Task 3: Kernel Seeding 与 Worker Sandbox 集成测试

**Files:**
- `packages/core/kernel/index.ts` — 引导加载外部 ZIP 插件
- `packages/plugins/__tests__/quiz.test.ts` — Quiz 插件在 Worker 模式下的测试
- `packages/core/__tests__/worker-rpc.test.ts` — 跨边界 RPC 代理与 Event 转发的集成测试

**Description:**
- 在 `packages/core/kernel/index.ts` 中，检测 `dist/plugins/` 目录中的 `.zip` 文件：
  - 如果数据库中不存在（或者版本不同），读取为 Buffer，调用 `PluginHost.installPluginFromZip(buffer)` 写入 DB。
  - 将数据库中这两个外部插件的 `execution_mode` 更新为 `'worker'`，`status` 为 `'active'`。
  - 在 Kernel 启动时调用 `PluginHost.activatePlugin(id)`。
- 在 `packages/plugins/__tests__/quiz.test.ts` 中：
  - 加载构建出的 ZIP 插件包。
  - 启动 Worker 模式激活。
  - 验证 RPC 传输，发送 `quiz.create` 成功调度主线程 `whiteboard.draw`。
- 在 `packages/core/__tests__/worker-rpc.test.ts` 中：
  - 测试沙箱崩溃隔离、RPC 参数权限拦截、以及主线程事件能被 Worker 监听到的特性。

**Verification:**
- 运行 `npx vitest run packages/plugins/__tests__/quiz.test.ts` 和 `npx vitest run packages/core/__tests__/worker-rpc.test.ts` 均成功。
- 全量测试全部通过。

## Verification Criteria

1. Quiz 和 Roll Call 插件已成功从 server.ts 中剥离，创建了独立的 TS 源码和 manifest.json。

3. `package.json` 中的 build pipeline 包含了对 `build-plugins.mjs` 的调用。

6. 新增测试通过，全量测试全部通过。
