# Plan: 08-01 — System Core Built-in Plugins Migration

**Objective:** 重构底层的 VFS 与 Process 插件至标准 ESM + Token DI 规范，声明 `IDatabaseToken` 使插件能通过 DI 访问 SQLite 数据库，并让 Kernel 在启动时通过 `PluginHost` 加载与激活这些特权内置插件。

## Tasks

**Files:**
- `packages/core/di/interfaces.ts` — 导入 better-sqlite3.Database 类型并导出 `IDatabaseToken`

### Task 4: Kernel 启动集成与系统插件自动加载

**Files:**
- `packages/core/kernel/index.ts` — 引入并自动加载系统内置插件，插入 DB 记录，支持硬崩溃保障。

**Description:**
- 在 Kernel 中引入 `VfsPlugin` 与 `ProcessPlugin`。
- 新增 `bootstrapSystemPlugins()` 私有方法并在构造函数末尾调用：
  ```typescript
  private async bootstrapSystemPlugins() {
    const systemPlugins = [
      { id: 'vfs-plugin-id', mod: VfsPlugin, name: 'Virtual File System Plugin' },
      { id: 'process-plugin-id', mod: ProcessPlugin, name: 'Background Process Plugin' }
    ];

    for (const plugin of systemPlugins) {
      try {
        // 1. 检查或插入到 SQLite plugins 表
        let row = this.db.prepare('SELECT id FROM plugins WHERE name = ? OR id = ?')
          .get(plugin.name, plugin.id);
        
        let actualId = row?.id;
        if (!row) {
          actualId = uuidv7(); // 或是预设固定 uuid
          this.db.prepare(
            'INSERT INTO plugins (id, name, manifest, source_code, status, created_at, loader_version, execution_mode) VALUES (?, ?, ?, ?, ?, ?, ?, ?)'
          ).run(
            actualId,
            plugin.name,
            JSON.stringify(plugin.mod.manifest),
            `// System built-in inline plugin: ${plugin.name}`,
            'installed',
            Date.now(),
            'esm',
            'inline'
          );
        }

        // 2. 注入模块定义到 ESM Loader（使其不需要从 DB source_code 编译加载）
        // 或是直接通过 PluginHost 内部 of pluginInstances 映射注入 (避开 EsmLoader 磁盘/网络查找)
        // 设计：为了兼容性，也可以将 builtin 插件直接注册到 pluginInstances
        // 并通过 PluginHost.activatePlugin(actualId) 激活它们
        // ... (在 PLAN 执行阶段设计最轻量的注册方案)
      } catch (err) {
        // VFS, Process 失败是 Hard crash 致命错误，直接抛出，导致内核启动失败
        console.error(`[Kernel] Failed to bootstrap critical system plugin ${plugin.name}:`, err);
        throw err;
      }
    }
  }
  ```
- 修改 `PluginHost` 的 `activatePlugin`，如果目标插件是内置插件且已内存加载，直接跳过 `EsmLoader.load`，从而支持无缝 Inline 运行。

**Verification:**
