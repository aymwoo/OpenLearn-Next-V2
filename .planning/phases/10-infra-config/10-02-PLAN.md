<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 宿主与远程子应用加载边界 | 宿主应用在运行时通过浏览器加载远程子应用的 remoteEntry.js 脚本，需要保证加载源的完整性与安全性，防范跨站脚本注入 (XSS) 风险。 |
| 第三方依赖与构建系统边界 | 使用 pnpm 安装外部 npm 包时，存在供应链投毒及包被恶意篡改的风险。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | remoteEntry.js | mitigate | 生产环境中将子应用静态构建产物与宿主独立托管在同一域名 Origin 的独立子路径下，通过本地静态代理和 HTTPS 加密传输，防范脚本在传输链路中被劫持或篡改（per D-11）。 |
| T-10-02 | Elevation of Privilege | SQLite Database &amp; API | mitigate | 在宿主后台插件注册表中，严格对插件 Entry URL 进行域名白名单正则校验与加签机制，限制非受信任域名的远程脚本执行（per D-10）。 |
| T-10-SC | Tampering | npm installs | mitigate | 对新增的微前端包依赖（@module-federation/* 等）执行 slopcheck 合法性审计，并通过人工 Blocking Checkpoint（Task 1）审核以防范冒充包投毒。 |
</threat_model>
