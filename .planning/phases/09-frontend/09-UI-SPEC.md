## Spacing Scale

Declared values (multiples of 4, Tailwind default scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, button padding (p-2) |
| md | 12px | Nav button padding (p-3), card inner gaps (gap-3) |
| lg | 16px | Default element spacing (p-4), card padding |
| xl | 24px | Section padding (p-6), layout gaps (gap-6) |
| 2xl | 32px | Major section breaks |
| 3xl | 48px | Page-level spacing |

Exceptions:
- `md: 12px` — maps to Tailwind `p-3`, required for existing nav button padding (App.tsx line 5451) and plugin card inner gaps (line 6330). Cannot change without altering existing UI that Phase 9 integrates with.

**Source:** Derived from existing `App.tsx` spacing patterns (lines 5450-5499 nav, lines 6295-6375 plugin center). Phase 9 new components use identical values.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (font-normal) | 1.5 (leading-relaxed) |
| Label | 12px (text-xs) | 700 (font-bold) | 1.33 |
| Heading | 18px (text-lg) | 700 (font-bold) | 1.25 |
| Display | 24px (text-xl) | 700 (font-bold) | 1.2 |

**Source:** Derived from existing `App.tsx` text patterns: body at `text-sm` (nav buttons, plugin descriptions), labels at `text-xs font-bold` (badges, status tags), headings at `text-lg font-bold` (section headers), display at `text-xl font-bold` (plugin card titles). Font weights reduced to 2 (400 + 700) per design system constraint; 12px label size provides sufficient visual distinction without a separate semibold weight.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF (white) | Page background, card surfaces, modal backgrounds |
| Secondary (30%) | #F9FAFB (gray-50) | Sidebar navigation, card headers, section backgrounds |
| Accent (10%) | #4F46E5 (indigo-600) | Active tab indicators, primary buttons, plugin card icons, link text |
| Destructive | #DC2626 (red-600) | Delete plugin button, destructive action confirmation |

Accent reserved for:
- Navigation tab active state (bg-indigo-50 text-indigo-700)
- Primary action buttons (bg-indigo-600 text-white on hover)
- Plugin card icon containers (bg-indigo-50 text-indigo-600)
- Extension Point slot icons
- LoadingSkeleton pulse color
- ZIP drop zone active border highlight

**Source:** Existing palette extracted from `App.tsx`: indigo used as primary accent across nav (line 5451), plugin center (line 6330), upload buttons (line 6400). Red used for delete buttons (line 6348). Phase 9 introduces no new semantic colors.

## Interaction States

### ExtensionPointRenderer States

| State | Visual | Trigger |
|-------|--------|---------|
| Loading | `<LoadingSkeleton />` -- pulsing gray placeholder | `React.lazy` is resolving the dynamic import |
| Loaded | Rendered plugin component | import() resolved successfully |
| Error | Error boundary fallback: `bg-red-50 border border-red-200 rounded-lg p-4 text-sm text-red-700` with message "Extension failed to load" and a "Retry Load" button | import() rejected or component threw during render |
| Empty | Nothing rendered (no extensions registered for slot) | `getExtensions(slot)` returns empty array |

### ZIPDropZone States

| State | Visual | Trigger |
|-------|--------|---------|
| Idle | Dashed gray border, upload icon + prompt text | No drag activity |
| Drag over | Dashed indigo border, indigo-tinted background, icon pulses | User drags a file over the zone |
| Processing | Spinner overlay, text changes to "Analyzing..." | File dropped, jszip is parsing |
| Error | Red border flash + error text below zone | ZIP parse failed or no manifest.json found |
| Success | Green border flash + brief "Ready" checkmark, then resets | ZIP parsed successfully, manifest preview shown |

### LegacyPluginBadge States

| State | Visual | Trigger |
|-------|--------|---------|
| Visible | Amber badge shown | `plugin.execution_mode === 'legacy'` |
| Hidden | Badge not rendered | `plugin.execution_mode !== 'legacy'` |
| Hover tooltip | Browser native tooltip: "This plugin uses the old execution format. A new-format version may be available." | Mouse hover over badge |

### MigrationPrompt States

| State | Visual | Trigger |
|-------|--------|---------|
| Visible | Amber banner shown at top of Developer tab | At least one plugin has `execution_mode === 'legacy'` |
| Hidden | Banner not rendered | No legacy plugins detected |
| Dismissed | Banner removed from DOM until next page load | User clicks dismiss (X) button on banner |
