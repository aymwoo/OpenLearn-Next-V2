<objective>
Establish a robust context bridging layer (MfeContext) between the Host Shell App and Module Federation remote applications. This enables child micro frontends (MFE) to subscribe to the host's central Zustand state (appStore), resolve host services safely via a whitelist-restricted DI container proxy (MfeServiceRegistryProxy), and publish/subscribe to real-time events via a reference-counted and leak-free Socket-EventBus bridge (MfeEventBusWrapper).

Purpose: Fulfills requirements MFE-BRIDGE-01 through MFE-BRIDGE-04. Connects remote view modules (whiteboard, courseware) to the host infrastructure without sacrificing host sandboxing safety or causing memory leaks.
Output: State store in src/store/appStore.ts, updated types.ts, MfeContextProvider logic, MfeLoaderCore bridge hookups, custom subscription hook useMfeEvent, and the test suite src/mfe/__tests__/bridge.test.tsx.
</objective>

<threat_model>
## Trust Boundaries
The following trust boundaries exist for Phase 12:

| Boundary | Description |
|----------|-------------|
| Host Context -> MFE Runtime | Host context is passed across the Module Federation boundary to the remote's `createMfeApp` entrypoint. Remotes run in the same browser thread but should only access exposed API surfaces. |
| MFE Runtime -> ServiceRegistry | Remotes request platform services. The boundary prevents unauthorized access to private host systems. |
| MFE EventBus -> Network (Socket.IO) | MFE triggers network broadcast. The boundary restricts formatting and authenticates payload delivery. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Elevation of Privilege | ServiceRegistry Resolution | mitigate | **DI Whitelist Enforcement**: `MfeServiceRegistryProxy` blocks calls to non-whitelisted private host services. Write methods (`register`, `unregister`) are excluded from the proxy interface to prevent remote apps from registering malicious services or modifying existing bindings. Compliant with ASVS Level 1. |
| T-12-02 | Spoofing | EventBus Publishing | mitigate | **Event Source Verification**: `MfeEventBusWrapper` intercepts all local events published by remotes and overwrites the `source` field with the remote's verified plugin name (e.g. `mfe-whiteboard`), preventing spoofed events from triggering unauthorized actions. |
| T-12-03 | Information Disclosure | Zustand Store Exposure | mitigate | **State Minimization & Isolation**: Only the central Zustand store instance containing non-sensitive shared layouts/business records (lessons, classes, user profile) is shared with the remotes. No database credentials, session tokens, or private administration properties are stored in this Zustand state. |
| T-12-04 | Tampering | server:* Event Dispatching | mitigate | **Backend Defense Gating**: All network messages published by remotes (`server:*` events) are intercepted and pushed to the backend via Socket.IO. The backend uses the `CapabilityGuard` and user session parsing to validate each message before processing or broadcasting. |
</threat_model>
