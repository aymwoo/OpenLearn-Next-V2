### Task 3: 创建 HotReloadController + debounce

**Files:**
- `packages/core/plugin-host/hot-reload.ts` — HotReloadController 类（同文件，在 FileWatcher 之后）

**Description:**
创建 HotReloadController 类，编排文件监听和 reload 触发：

```typescript
export class HotReloadController {
  private fileWatcher: FileWatcher;
  private debounceTimers = new Map<string, NodeJS.Timeout>();
  private readonly DEBOUNCE_MS = 300;
  private enabled = false;

  constructor(
    private pluginHost: PluginHost, // 用于调用 reloadPlugin
    private watchDir: string,
  );

  // 启动热重载监听（仅 dev 模式调用）
  async start(): Promise<void>;

  // 停止热重载监听
  async stop(): Promise<void>;

  // 注册插件到文件监听
  registerPlugin(pluginId: string, filePath: string): void;

  // 处理文件变更事件
  private handleFileChange(pluginId: string, filePath: string): Promise<void>;
}
```

核心逻辑:
- `start()`: 检查 `NODE_ENV === 'development'`，不是则跳过。然后调用 `fileWatcher.startWatch()`
- `handleFileChange()`:
  1. 清除该 pluginId 的旧 debounce timer
  2. 创建新 timer (300ms)
  3. Timer 到期后，读取文件内容，调用 `pluginHost.reloadPlugin(pluginId, newSourceCode)`
  4. 捕获错误: 不向上抛（保持监听器存活），console.error 报告
- `stop()`: 清除所有 debounce timer，调用 `fileWatcher.stopWatch()`

**Verification:**
- NODE_ENV=production 时 start() 是 no-op
- 同插件 300ms 内多次变更只触发一次 reload
- handleFileChange 异常不崩溃监听器
- stop() 后不再触发 reload
