### 委托映射（PluginRuntime → PluginHost）

| PluginRuntime 方法 | 委托给 | 说明 |
|-------------------|--------|------|
| loadedPlugins (getter) | 保留现有实现 + 从 PluginHost 合并 | 不期望行为改变 — 仅从 DB 查询 |
| loadFromDB() | 保留现有实现 + 委托给 PluginHost.restoreActivePlugins() | ESM 插件由 PluginHost 恢复；vm 插件由 PluginRuntime 的 evaluateAndActivate 恢复 |
| installPlugin(sourceCode) | 保留现有名称 + 委托给 pluginHost.installPlugin(sourceCode) | 返回 manifest 以保持调用方兼容 |
| installPluginFromZip(zipBuffer) | 委托给 pluginHost.installPluginFromZip(zipBuffer) | 返回 manifest 以保持调用方兼容 |
| togglePlugin(id) | 委托给 PluginHost | 检查状态 → 停用或激活 |
| uninstallPlugin(id) | 委托给 pluginHost.uninstallPlugin(id) | DB 清理由 PluginHost 处理 |
| deactivatePlugin(id) (私有) | 委托给 pluginHost.deactivatePlugin(pluginId) | pluginId 格式在 PluginRuntime 中为 uuid，在 PluginHost 中也保持一致 |
| evaluateAndActivate (私有) | 保留不变 | 仅限 vm 路径 — 在 Phase 8 之前不碰 |
| evaluateAndActivateEsm (私有) | 保留但简化为委托给 PluginHost.activatePlugin | ESM 加载逻辑现在在 PluginHost 中 |

### 兼容性约束（D-13）
- 公共方法签名必须与当前 `plugin-runtime/index.ts` 完全一致
- 返回类型不加宽也不收窄 — 即使返回 `any` 也不能改为 `Promise<void>`
- server.ts 中的调用者（`POST /api/plugins/install`、`POST /api/plugins/toggle`、`DELETE /api/plugins/:id`）不能看到中断性变更
