| Boundary | Description |
|----------|-------------|
| PluginContext.services → Plugin | 插件代码通过冻结的 services 对象访问内核服务。每个服务方法通过 createSafeFunction 包装，切断原型链和构造函数访问 |
| ServiceRegistry → PluginHost | PluginHost 通过 ServiceRegistry.resolve() 获取原始服务实例 — 包装器层增加安全约束 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-04-08 | Spoofing | context-builder.ts | mitigate | manifest.id 通过 PluginHost 传递，由 manifestSchema.parse() 验证。插件无法伪装为其他插件 — actorId 来自 manifest，而非插件代码 |

| T-04-10 | Information Disclosure | context-builder.ts | mitigate | ctx.services 被 Object.freeze() 冻结，防止插件替换或添加服务引用。每个插件获得独立的包装服务实例（无跨插件服务泄露） |
