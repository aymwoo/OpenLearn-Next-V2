    **方法 2: activatePlugin(pluginId: string): Promise<void>**（D-10, D-11, D-12）：
    遵循 RESEARCH.md lines 446-514 的精确数据流：
    1. 获取 currentState，验证转换 INSTALLED/INACTIVE/ERROR → ACTIVATING
    2. 设置状态为 ACTIVATING
    3. 从 DB 加载插件：`SELECT source_code, manifest FROM plugins WHERE id = ?`
    4. `await this.esmLoader.load(row.source_code)` → PluginModule
    5. 从模块导出提取 plugin（`mod.default ?? mod`）、manifest（`plugin.manifest ?? mod.manifest`）、activate（`plugin.activate ?? mod.activate`）
    6. 如果缺少 manifest 或 activate，则抛出 EsmActivationError
    7. `manifestSchema.parse(manifest)` 验证 schema
    8. `await buildContext(this.serviceRegistry, this.resourceTracker, pluginId, manifest)` 构建 PluginContext
    9. 通过 capabilityService 授予能力，actorId 格式：`plugin:${manifest.id}`
    10. `await Promise.race([activate(ctx), timeout(5000)])` — 在超时时 reject（EsmLoadTimeoutError）
    11. 成功：设置状态为 ACTIVE，存储 pluginInstances（manifest、activate、deactivate），DB UPDATE status='active'
    12. **失败 catch（D-12）**：设置状态为 ERROR，调用 `this.resourceTracker.disposeAll(pluginId)` 回滚，通过 capabilityService.revokeAll 撤销能力。通过 `throw err` 重新抛出原始错误（D-10：不影响其他插件，调用者处理记录）

    **方法 3: deactivatePlugin(pluginId: string): Promise<void>**（D-09, D-11）：
    遵循 RESEARCH.md lines 522-548 的精确数据流：
    1. 获取 currentState — 如果 UNINSTALLED 或未找到，静默返回
    2. 验证转换 ACTIVE → DEACTIVATING
    3. 设置状态为 DEACTIVATING
    4. 获取 plugin 实例（来自 this.pluginInstances）
    5. 如果实例有 deactivate 函数：`await Promise.race([instance.deactivate(), timeout(5000)])`
    6. **finally 块**：无论成功、超时还是 deactivate 错误 — 始终执行：`this.resourceTracker.disposeAll(pluginId)` → 设置状态为 INACTIVE → DB UPDATE status='inactive' → 通过 capabilityService.revokeAll 撤销能力
    7. 在 catch 和 finally 中：deactivate 超时或错误使用 `console.error` 记录，**不重新抛出**（停用必须是 best-effort）

    **方法 4: uninstallPlugin(pluginId: string): Promise<void>**：
    1. 如果是 ACTIVE：调用 `await this.deactivatePlugin(pluginId)`（必须先停用）
    2. 验证转换 INACTIVE/ERROR/INSTALLED → UNINSTALLED
    3. 从 DB DELETE：`DELETE FROM plugins WHERE id = ?` + `DELETE FROM plugin_storage WHERE plugin_id = ?`（使用 manifest.id 作为 plugin_id — 匹配现有 schema）
    4. 清理内存：`this.pluginStates.delete(pluginId)` + `this.pluginInstances.delete(pluginId)`

    **方法 5: installPluginFromZip(zipBuffer: Buffer): Promise<Manifest>**（从 PluginRuntime lines 69-107 迁移）：
    - 如果未配置 this.esmLoader 则抛出
    - 调用 `validateAndBundleZip(zipBuffer)` 获取 `{ manifest, bundledCode }`
    - 生成 uuidv7() 作为 id
    - 调用 `ensureUniqueManifestId(manifest.id)`
    - INSERT 到 DB（含 loader_version='esm'、zip_package=BLOB）
    - 保存为 INSTALLED 状态
    - 返回 manifest（与调用方期望兼容 — 与 PluginRuntime 的返回类型匹配）
    - 失败时清理 DB 条目

    **方法 6: restoreActivePlugins(): Promise<void>**（从 PluginRuntime.loadFromDB 迁移）：
    - 查询 `SELECT * FROM plugins WHERE status = 'active' AND loader_version = 'esm'`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plugin source code → PluginHost | 不受信任的插件代码由 EsmLoader 加载。activate/deactivate 调用超时保护（5 秒）。原型链通过 createSafeFunction 切断 |
| PluginHost → SQLite | 用于插件存储和恢复的 DB 访问。SQL 参数化（`db.prepare(...).run(params)`）防止注入 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
