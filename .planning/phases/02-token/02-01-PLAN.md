文件头注释：说明 StorageService 是内核级 KV 持久化存储，使用 `'__kernel__'` 作为 plugin_id 命名空间，per-plugin 隔离由 PluginRuntime 的 wrappedStorage 层提供。

- `import type { IStorageService } from './interfaces.js'`

- 构造函数：`constructor(private db: BetterSqlite3.Database) {}` — 从 DI 模式学习：与 ProcessManager( Kernel ) 相同的构造函数注入模式
- `async get(key: string): Promise<unknown>` — 执行 `SELECT value FROM plugin_storage WHERE plugin_id = '__kernel__' AND key = ?`，用 `JSON.parse(row.value)` 解析，无结果返回 `null`
- `async set(key: string, value: unknown): Promise<void>` — 执行 `INSERT INTO plugin_storage (plugin_id, key, value, updated_at) VALUES ('__kernel__', ?, ?, ?) ON CONFLICT(plugin_id, key) DO UPDATE SET value = excluded.value, updated_at = excluded.updated_at`，value 通过 `JSON.stringify(value)` 序列化，updated_at 使用 `Date.now()`
- `async delete(key: string): Promise<void>` — 执行 `DELETE FROM plugin_storage WHERE plugin_id = '__kernel__' AND key = ?`

注意：不包含 try-catch 包装——error 冒泡到调用方处理。不使用 `createSafeFunction`（这是 PluginRuntime 的安全层，不属于 IService 实现）。

  1. 从 `ai_providers` 表查找有 API key 的 provider：`SELECT id, name, api_url, api_key, model_name FROM ai_providers WHERE api_key IS NOT NULL AND api_key != '' LIMIT 1`

  5. 如两者都无：`throw new Error('No AI providers or Gemini API key configured in the system.')`
- 注意：不在 AIService 中添加 `console.error` 或 try-catch——这属于 PluginRuntime 包装层。AIService 是纯粹的服务实现，异常原样抛出。
