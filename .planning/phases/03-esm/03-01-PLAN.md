<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifestSchema.parse() — 不受信输入 | manifest.json 字符串来自用户上传的 ZIP 包，在安装阶段通过 zod schema 校验。schemaless/畸形 JSON 在 parse() 阶段被拒绝，ZodError 包含精确字段路径 |
| EsmLoader.load() — 不受信输入 | code 字符串来自用户代码或 ZIP 包提取内容。EsmLoader 只返回模块导出，不暴露文件系统路径（data: URL 的 import.meta.url 是 data: URL 本身） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | EsmLoader.load() | accept (Phase 3 不涉及) | ESM 模块在 data: URL 环境中无法访问 require/fs/child_process。EsmLoader 只返回模块导出，PluginRuntime 侧保留 wrapped* 安全包装器（D-03）。esbuild external 策略在 Plan 04 实施 |
| T-03-02 | Denial of Service | manifestSchema.parse() | mitigate | zod 自动限制字符串长度（默认为安全范围）。manifest.json 文件大小需在解压前检查（限制 64KB，Plan 04 实施）。无效 manifest 在 parse() 阶段快速失败 |
| T-03-03 | Tampering | manifest.json 内容注入 | mitigate | D-10: zod schema 在安装时校验所有必需字段 id/name/version/main 非空。manifest 缺少必需字段在 parse() 阶段被拒绝，阻止安装。Pitfall 3 已处理（不使用 safeParse） |
| T-03-04 | Information Disclosure | EsmLoaderError 错误消息 | mitigate | 错误消息使用 [EsmLoader] 标签进行结构化记录，不包含文件系统路径、堆栈跟踪或源代码内容。data: URL hash 在日志中隐藏明文代码 |
| T-03-05 | Repudiation | 插件安装审计 | accept (Phase 3 不涉及) | 现有 eventBus 审计日志捕获所有插件操作事件。Phase 3 不改变事件发布模式 |
| T-03-06 | Elevation of Privilege | EsmLoaderError 类伪造 | mitigate | 所有错误类继承自 Error，使用 this.name 明确标识类型。PluginRuntime 通过 instanceof 检查精确捕获特定错误类型 |

</threat_model>
