<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| NodeEsmLoader.load() — data: URL import() | 不受信代码字符串通过 base64 编码为 data: URL 后 import()。Node.js ESM loader 的 data: URL 模块无法访问 require/fs/child_process。import.meta.url 是 data: URL 本身，不泄露文件系统路径 |
| BrowserEsmLoader.load() — Blob URL import() | 不受信代码字符串通过 Blob URL import()。浏览器 sandbox 自然隔离。URL.revokeObjectURL() 在 finally 中清理防止内存泄漏 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | NodeEsmLoader.load() | mitigate | data: URL 模块在 Node.js 中无法访问 require/fs。import() 只能加载 ESM 模块，不会执行 CJS require。base64 编码防止 URL 注入攻击代码 |
| T-03-04 | Information Disclosure | data: URL import.meta.url | accept | import.meta.url 暴露 data: URL 内容（包含 base64 编码的源代码）。这是 data: URL 方案的固有属性——无法避免。源代码本身已在安装时由管理员上传，不算泄露。文件系统路径不暴露 |
| T-03-06 | Denial of Service | 无限循环代码 | mitigate | NodeEsmLoader.load() 依赖 PluginRuntime 的 Promise.race 超时（D-14，Plan 04 实施）。BrowserEsmLoader 由浏览器 Tab 级隔离，主线程阻塞只影响当前 Tab |
| T-03-07 | Information Disclosure | BrowserEsmLoader Blob URL | mitigate | Blob URL 仅在创建它的 origin 可访问。URL.revokeObjectURL() 在 finally 中立即清理，防止 URL 悬挂 |

</threat_model>
