# Implementation Plan - Task 12-01-02

## Goal
Migrate 9 states from `src/App.tsx` local state to the central vanilla Zustand store `src/store/appStore.ts` and wrap the rendering container with `<MfeContextProvider>`.

## Proposed Changes

### 1. `src/App.tsx` Imports & Globals
- Import `useAppStore`, `appStore`, `MfeContextProvider`, and `EventBus`.
- Instantiate a global `hostEventBus` outside of the `App` component.

### 2. `src/App.tsx` Component State
- Remove local `useState` hook definitions for the 9 states:
  - `lang`
  - `session`
  - `lessons`
  - `selectedLesson`
  - `elements`
  - `classes`
  - `students`
  - `liveClassSelectedClassId`
  - `liveClassIsActive`
- Replace them with Zustand selectors/actions:
  ```typescript
  const lang = useAppStore((s) => s.lang);
  const setLang = useAppStore((s) => s.setLang);
  // and so on...
  ```

### 3. `src/App.tsx` Render Wrap
- Wrap the main return container in `App` component with:
  ```typescript
  <MfeContextProvider value={{ store: appStore, eventBus: hostEventBus, serviceRegistry: host.getRegistry() }}>
    ...
  </MfeContextProvider>
  ```

## Verification
- Run `npm run lint` (runs `tsc --noEmit`) to verify no compilation or type errors.
- Run `npx vitest run src/mfe/__tests__/bridge.test.tsx` to verify Zustand State Sync test passes.
