# Implementation Plan - Task 332ed193 (Fix Database Path in Production Bundle)

## Goal
Fix the PM2 startup error `TypeError [ERR_INVALID_ARG_TYPE]: The "path" argument must be of type string or an instance of URL. Received undefined` caused by `fileURLToPath(import.meta.url)` in `packages/core/db/index.ts` when running in a bundled CommonJS environment.

## Root Cause
- In ESM mode (e.g. tsx development), `import.meta.url` is a valid string, so `fileURLToPath(import.meta.url)` successfully gets the filename.
- When built via `esbuild` into `dist/server.cjs` (CommonJS), `import.meta.url` evaluates to `undefined`, which throws a `TypeError` when passed to `fileURLToPath()`.

## Proposed Changes

### 1. Update `packages/core/db/index.ts`
- Implement a runtime check to verify if `import.meta` and `import.meta.url` are defined.
- If defined, resolve `educational_os.db` relative to `import.meta.url`.
- If not defined (such as when bundled as CommonJS inside `dist/server.cjs`), resolve it relative to `__dirname` pointing to `../packages/core/db/educational_os.db`.

```typescript
let dbPath: string;
if (typeof import.meta !== 'undefined' && import.meta.url) {
  const __filename = fileURLToPath(import.meta.url);
  const __dirname = path.dirname(__filename);
  dbPath = path.join(__dirname, 'educational_os.db');
} else {
  // Bundled CommonJS environment: __dirname is the directory of dist/server.cjs (i.e. root/dist)
  dbPath = path.join(__dirname, '../packages/core/db/educational_os.db');
}
```

## Verification
1. Run `pnpm run build` to build the production server bundle.
2. Run `node dist/server.cjs` to verify that the server starts up without throwing the `TypeError` and initializes the database successfully.
3. Run `pnpm run test` or specific core database-dependent tests (such as `packages/core/__tests__/kernel-plugins.test.ts`) to ensure development environment compatibility.
